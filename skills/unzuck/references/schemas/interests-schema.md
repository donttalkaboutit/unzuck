# interests.json Schema

This file defines the user's content interest profile. It drives scoring and platform configuration.

## Full schema

```json
{
  "description": "Brief natural-language description of the user's interests and role",
  "topics": [
    {
      "name": "AI/ML",
      "weight": 0.9,
      "keywords": ["LLM", "transformer", "fine-tuning", "GPT", "Claude", "diffusion"]
    },
    {
      "name": "Systems programming",
      "weight": 0.7,
      "keywords": ["Rust", "C++", "memory management", "kernel", "performance"]
    },
    {
      "name": "Startups",
      "weight": 0.5,
      "keywords": ["YC", "fundraising", "seed round", "product-market fit"]
    }
  ],
  "negativeTopics": [
    {
      "name": "Crypto trading",
      "keywords": ["altcoin", "memecoin", "token launch", "pump"],
      "penalty": -30
    },
    {
      "name": "Celebrity gossip",
      "keywords": ["kardashian", "paparazzi", "red carpet"],
      "penalty": -30
    }
  ],
  "platformOverrides": {
    "hackernews": { "maxItems": 25 },
    "youtube": { "maxItems": 10 },
    "twitter": { "maxItems": 10 },
    "linkedin": { "maxItems": 5 },
    "facebook": { "maxItems": 0 },
    "instagram": { "maxItems": 0 }
  },
  "history": {
    "deduplicationDays": 7,
    "file": ".content-curator/history.json"
  },
  "createdAt": "2026-03-23T00:00:00Z",
  "autoGenerated": false
}
```

## Field reference

| Field | Required | Description |
|-------|----------|-------------|
| `description` | Yes | Natural-language summary of who the user is and what they care about |
| `topics` | Yes | Array of interest topics with weights (0.0–1.0) |
| `topics[].name` | Yes | Human-readable topic name |
| `topics[].weight` | Yes | Importance weight: 0.9 = very important, 0.3 = minor interest |
| `topics[].keywords` | No | Specific terms that indicate this topic. Helps with matching. |
| `negativeTopics` | No | Topics to suppress. Items matching these get a score penalty. |
| `negativeTopics[].penalty` | No | Score adjustment (negative number). Default: -30 |
| `platformOverrides` | No | Per-platform maxItems. 0 = skip platform. |
| `history` | No | Dedup config. `deduplicationDays` controls how long to remember seen URLs. |
| `autoGenerated` | No | `true` if profile was bootstrapped from feeds rather than manually set |
| `topicStats` | No | Per-topic match counts from recent runs (last 7). Used by Step 5.5 to detect stale/emerging topics. |
| `topicStats[topic].matchCounts` | No | Array of integers — how many scored items matched this topic in each of the last 7 runs. |
| `profileUpdates` | No | Audit log of automatic profile changes. Each entry has `date`, `action`, `topic`, and `reason`. |
| `workingDirectory` | No | Absolute path to the directory where digests and profile are stored. Set during onboarding. |

## Weight guidelines

- **0.8–1.0**: Core professional/passion interest — always want to see this
- **0.5–0.7**: Solid interest — happy to see it but don't need everything
- **0.2–0.4**: Casual interest — only show if very high quality/engagement
- **< 0.2**: Barely interested — almost never show

## Bootstrap output

When auto-generating from feed content, set `autoGenerated: true` and derive topics from:
- YouTube topic chips and channel themes
- Twitter/X content themes and "Who to follow" suggestions
- LinkedIn job title and professional content themes
- HN front-page topic distribution
- Facebook/Instagram group and page themes
